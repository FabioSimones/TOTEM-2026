# Visão Geral do Produto

## Resumo

O sistema consiste em uma solução de autoatendimento para fast foods, onde o cliente pode realizar pedidos diretamente em um tablet, totem ou tela touch.

O cliente poderá:

- Visualizar o cardápio
- Escolher produtos
- Personalizar itens
- Selecionar forma de pagamento
- Receber senha ou comprovante de retirada
- Acompanhar o status do pedido

## Módulos principais

O sistema será composto por:

1. Totem do cliente
2. Backend Java/Spring Boot
3. Banco de dados PostgreSQL
4. Painel do caixa
5. Painel da cozinha
6. Painel administrativo
7. Módulo de segurança
8. Módulo de dispositivos
9. Módulo de pagamento
10. Auditoria

## Problema que resolve

Em operações de fast food, filas no caixa, dependência total de atendentes e falhas de comunicação podem gerar lentidão, pedidos incorretos e perda de produtividade.

O totem reduz a sobrecarga do caixa e padroniza a coleta do pedido.

## Benefícios para o cliente

- Autonomia para escolher produtos
- Visualização clara do cardápio
- Facilidade para personalizar pedidos
- Redução de tempo em filas
- Acompanhamento do pedido por senha

## Benefícios para o restaurante

- Organização do fluxo de pedidos
- Redução de sobrecarga no caixa
- Separação entre pedido, pagamento e preparo
- Controle de pedidos pagos e pendentes
- Gestão dinâmica do cardápio sem alteração no código
- Possibilidade de expansão para múltiplos dispositivos

## Objetivo principal

Criar uma solução funcional de autoatendimento para fast food, permitindo que o cliente realize pedidos de forma independente e que o restaurante controle pagamento, preparo, retirada e cardápio.
